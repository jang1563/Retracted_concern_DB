# Adjudication Protocol Overview

The operational adjudication pack is generated into each release directory as `adjudication_protocol.md`, but this file records the standing review policy.

## Review Goals

- Validate weak labels
- Validate issue tags
- Review public-site wording for caution and factuality

## Minimum Double-Review Fields

- reviewer A decision
- reviewer B decision
- consensus decision
- wording review
- notes

## Eligibility Labels

- `official_notice`
- `curator_review`
- `none_known_at_snapshot`

`none_known_at_snapshot` is used for background/negative adjudication rows with no public integrity signal at the snapshot. It should not be treated as a curator-gated allegation.

## Release Rule

- The adjudication subset is intended for internal evaluation and manuscript support
- Public release eligibility should not be escalated solely from weak external signals without curator review
